# Repository Instructions

This repository receives PRD/DesignDoc generated by an n8n workflow and the implementation patch produced by an AWS Bedrock (Claude Sonnet) Lambda implementer.

## General guidance

- Keep changes small and reviewable.
- Prefer Markdown documentation for coordination artifacts.
- Do not add OpenAI API keys or external implementation agent integrations.
- Implementation is delegated to the Bedrock Lambda invoked from the n8n workflow; pull requests created here are expected to be reviewed and merged by humans.

## Pull request expectations

A pull request opened by the workflow contains:

- A DesignDoc Markdown committed under `docs/design-docs/`.
- Source-code changes generated by the Bedrock implementer.
- A summary comment posted by the workflow that lists changed files, the model id used, test results, and any remaining risks.

Reviewers should:

- Verify that the DesignDoc reflects the requested change.
- Verify that the patch matches the DesignDoc and respects existing conventions.
- Run the project quality gates (formatter, linter, static analysis, tests, build) before merging.
